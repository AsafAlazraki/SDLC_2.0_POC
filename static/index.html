⚡ THE PIPELINE — 5 PHASES FROM URL TO VERDICT

Every analysis runs the same deterministic pipeline. Here's exactly what happens, in order.

0
🔍

Reconnaissance Pre-Pass

Gemini 2.0 Flash

Before the main fleet launches, a fast, cheap Gemini call scans the first 80,000 characters of the codebase and returns structured JSON: primary language, frameworks, databases, auth mechanism, CI/CD setup, complexity rating, entry points, and any immediately visible red flags.

This reconnaissance context is injected into every one of the 16 agents' prompts, so no agent wastes capacity rediscovering the tech stack. They all start with verified baseline facts and go straight to deep domain analysis. If recon fails, it degrades gracefully — the fleet launches anyway without it.

{"primary_language":"Python 3.11","frameworks":["FastAPI","Supabase"],"auth_mechanism":"API key (env-only)","red_flags":["No rate limiting on public endpoints"],"estimated_complexity":"Medium"}

Repository Ingestion

GitHub REST API

The engine fetches the full recursive file tree via the GitHub API, then downloads each file's raw content in parallel. No cloning, no git — pure API, works on any public repository.

Binary skip list PNG, JPG, SVG, PDF, WOFF, ZIP, EXE, DLL, SO and 20+ more extensions are skipped automatically
Directory skip list node_modules, .git, __pycache__, dist, build, venv, vendor — dependency noise removed before any agent sees it
Per-file size cap 200KB per file — prevents a single minified bundle from consuming the entire context budget
Total corpus cap 1,000,000 characters total — enough for very large repos while keeping costs predictable

Each file is emitted as a structured block: ====FILE: path/to/file.ext==== followed by raw content. This separator is what the next phase uses to slice and score files per persona.

Persona-Aware Context Filtering

Python — Zero API cost

Before any agent sees the codebase, filter_context_for_persona() re-parses the file blocks, scores each file's relevance to that specific agent's domain, sorts by score descending, and fills up to that agent's token budget with the highest-signal files.

This is the biggest quality improvement in the engine. Previously every agent received the same 60K-char head-truncation — the Security Engineer was wading through CSS variables, the UX Designer was reading database migrations. Now each agent receives a deep, targeted slice.

Parallel Fleet Execution

asyncio — Python

All 16 specialist agents are created as async tasks and run concurrently. Results are yielded as they arrive via asyncio.as_completed() — the fastest agent's report appears in the UI first, regardless of launch order. Results stream to the browser over Server-Sent Events (SSE) in real time. Each agent's prompt includes a forward-thinking technology mandate: they must consider AI tools, low-code alternatives, and automation alongside traditional approaches.

Rate Limit Safeguards (5-Layer Defence)
🔒
Semaphore (max 2 concurrent Claude calls) — Anthropic Tier 1 allows 30K input tokens/min. The semaphore ensures no more than 2 Claude agents are making API calls simultaneously, preventing burst overload.
⏱️
Staggered launch (4s between Claude agents) — Claude agents don't all fire at t=0. They launch one every 4 seconds, spreading the token burst across ~28 seconds rather than spiking it all at once.
🔄
Retry with exponential backoff — On 429 (rate limit) or 529 (overload), the agent waits 15s → 30s → 60s between up to 3 retry attempts before giving up.
✂️
Context truncation per persona — After relevance filtering, each Claude agent's context is capped at its persona-specific limit (50K–100K chars ≈ 12K–25K tokens), keeping individual prompt sizes within the rate limit window.
🔀
Graceful Gemini fallback — If a Claude agent exhausts all retries, it silently switches to Gemini with live Google Search grounding rather than returning an error. The user sees a completed report regardless.

Synthesis — "The Verdict"

Claude Sonnet 4.6 + Extended Thinking

After all 16 agents complete, a 17th sequential call reads every report concatenated and produces the master action plan. This is the only step that knows about all other agents — it resolves contradictions, identifies consensus (findings validated by 3+ independent agents), fills blind spots, and produces three distinct strategic paths forward: Conservative, Balanced, and Transformative — each with specific tools, timelines, and investment levels tailored to this codebase.

Extended Thinking is enabled with an 8,000-token private reasoning budget. Before writing a single word of the final report, Claude reasons through the contradictions in a private scratchpad — e.g., "Performance says cache aggressively; Security flags caching as a risk vector — which wins and why?" The thinking blocks are discarded before storing the output. Only the authoritative final answer is shown. Output budget is 10,000 tokens.

✨ BUILT-IN FEATURES

Everything is integrated — no manual export steps, no copy-pasting between tools.

💬
Agent Q&A Chat

Every report card has an "Ask" button. Opens a chat modal pre-seeded with that agent's full report as context. Ask follow-ups: "Why did you flag X?", "Give me the code for fix #3", "What's the priority order?" Each agent responds in character with its own domain expertise.

📤
GitHub Issues Export

The BA report's structured stories are parsed and exported directly to GitHub Issues in the analysed repository via the GitHub API. Each issue includes the user story, all acceptance criteria, story points, priority label, and technical notes — ready for sprint planning.

📊
Live Architecture Diagrams

The Architect agent produces a Mermaid.js diagram in its report. The frontend automatically detects, extracts, and renders it as an interactive SVG — no copy-pasting to Mermaid Live. The modernisation roadmap is also parsed into a visual 3-phase grid.

🗃️
Jira-Style Backlog

The BA report's user stories are parsed by regex and rendered as a kanban-style board grouped by priority (Must Have / Should Have / Could Have) with story points, acceptance criteria, and technical notes displayed inline — no external tool needed.

🕰️
Report Persistence & History

Every completed analysis is automatically saved to Supabase (PostgreSQL). The History view shows all past runs with repo URL, date, and client context. Clicking any history entry re-renders the full 16-card report from saved JSON — no re-analysis needed.

🏢
Client Context Injection

Create a client profile in the Admin panel (name, description, constraints). When selected, this context is injected into every agent's prompt — agents tailor recommendations to your client's industry, scale, risk profile, and regulatory environment.

📡 REAL-TIME STREAMING ARCHITECTURE

Results stream to your browser the moment each agent completes — you don't wait for all 15 to finish before seeing anything.

Backend (FastAPI)
status — Fleet lifecycle messages (cloning, launching, complete)
agent_update — Per-agent status (Analyzing Source Code... / Rate limited — waiting 15s...)
agent_result — Full report payload {persona, name, emoji, status, content}
error — Structured error with message
→
Frontend (Vanilla JS)
Progress bar + fleet status message updated
Per-agent card spinner and sub-status text updated live
Report card rendered with persona-specific renderer (Jira board, Mermaid diagram, or markdown)
Error displayed inline, fleet continues for other agents

The SSE connection uses sse-starlette on the backend and the browser's native EventSource API on the frontend. No WebSocket handshake overhead, no polling. The connection closes automatically when the synthesis result is yielded.
