    ⚡ THE PIPELINE — 8 PHASES FROM URL TO LIVING KNOWLEDGE

    Every analysis runs the same deterministic pipeline — but now with memory, confidence gating, and post-run learning. Here's exactly what happens, in order.

    0
    🔍

    Reconnaissance Pre-Pass

    Gemini 2.0 Flash

    Before the main fleet launches, a fast, cheap Gemini call scans the first 80,000 characters of the codebase and returns structured JSON: primary language, frameworks, databases, auth mechanism, CI/CD setup, complexity rating, entry points, and any immediately visible red flags.

    This reconnaissance context is injected into every one of the 16 agents' prompts, so no agent wastes capacity rediscovering the tech stack. They all start with verified baseline facts and go straight to deep domain analysis. If recon fails, it degrades gracefully — the fleet launches anyway without it.

    {"primary_language":"Python 3.11","frameworks":["FastAPI","Supabase"],"auth_mechanism":"API key (env-only)","red_flags":["No rate limiting on public endpoints"],"estimated_complexity":"Medium"}
    1
    📡

    Repository Ingestion

    GitHub REST API

    The engine fetches the full recursive file tree via the GitHub API, then downloads each file's raw content in parallel. No cloning, no git — pure API, works on any public repository.

    Binary skip list PNG, JPG, SVG, PDF, WOFF, ZIP, EXE, DLL, SO and 20+ more extensions are skipped automatically
    Directory skip list node_modules, .git, __pycache__, dist, build, venv, vendor — dependency noise removed before any agent sees it
    Per-file size cap 200KB per file — prevents a single minified bundle from consuming the entire context budget
    Total corpus cap 1,000,000 characters total — enough for very large repos while keeping costs predictable

    Each file is emitted as a structured block: ====FILE: path/to/file.ext==== followed by raw content. This separator is what the next phase uses to slice and score files per persona.

    1.5
    🧠

    Episodic Memory Loading

    Supabase — Zero API cost

    If this repository has been analysed before under a project, the engine loads episodic memory — a structured summary of what previous runs found, what changed, and what the agents learned. This is not raw history; it's distilled institutional knowledge.

    Episodic memory includes: the number of previous runs, per-agent findings summaries (truncated to 2K chars each), the last 3 synthesis verdicts, and all living documents (lessons learned, decision log, risk register, tech debt tracker). This context is injected into every agent's prompt so they don't repeat themselves — they build on prior knowledge.

    Project-scoped Memory is tied to projects, not repositories. The same repo analysed under different projects has separate memory tracks.
    Run history (last 5) Agent findings from the most recent 5 runs are summarised and injected — enough context without bloating prompts.
    Living documents 6 auto-generated documents (lessons learned, decision log, risk register, etc.) are loaded and made available to every agent.
    Graceful degradation If no project is selected or no prior runs exist, memory is empty — the fleet launches normally without it.

    🎯

    Persona-Aware Context Filtering

    Python — Zero API cost

    Before any agent sees the codebase, filter_context_for_persona() re-parses the file blocks, scores each file's relevance to that specific agent's domain, sorts by score descending, and fills up to that agent's token budget with the highest-signal files.

    This is the biggest quality improvement in the engine. Previously every agent received the same 60K-char head-truncation — the Security Engineer was wading through CSS variables, the UX Designer was reading database migrations. Now each agent receives a deep, targeted slice.

    Confidence Pre-flight Check

    Gemini 2.0 Flash

    Before the full fleet runs, each agent performs a fast self-assessment on a small sample (15K chars) of their filtered context. They return a structured confidence level — High, Medium, or Low — along with specific gaps in information and clarifying questions for the user.

    This is where the engine becomes interactive. The fleet pauses. You see each agent's confidence level, what they already know, what they're unsure about, and what questions they have. You can answer questions, provide additional context URLs, or skip straight to analysis. High-confidence agents' preliminary findings are automatically shared with struggling agents via cross-agent briefing.

    Confidence levels ● High — Agent has enough context. ● Medium — Could benefit from more info. ● Low — Significant gaps detected.
    Cross-agent briefing High-confidence agents share their preliminary findings with medium/low agents — like a senior colleague giving a heads-up before a meeting.
    User answers Your answers and any URLs you provide are injected directly into the asking agent's prompt as verified context. Agents skip guessing and use your ground truth.
    Optional gate You can skip the Q&A entirely — the fleet launches with whatever confidence it has. No delays if you're in a hurry.
    3
    🚀

    Parallel Fleet Execution

    asyncio — Python

    All 18 specialist agents — plus any custom specialists spawned from previous runs — are created as async tasks and run concurrently. Results are yielded as they arrive via asyncio.as_completed() — the fastest agent's report appears in the UI first, regardless of launch order. Results stream to the browser over Server-Sent Events (SSE) in real time. Each agent's prompt includes: the codebase context, recon findings, episodic memory, any user-provided answers from the confidence Q&A, cross-agent briefing context, and the forward-thinking technology mandate.

    Rate Limit Safeguards (5-Layer Defence)
    🔒
    Semaphore (max 2 concurrent Claude calls) — Anthropic Tier 1 allows 30K input tokens/min. The semaphore ensures no more than 2 Claude agents are making API calls simultaneously, preventing burst overload.
    ⏱️
    Staggered launch (4s between Claude agents) — Claude agents don't all fire at t=0. They launch one every 4 seconds, spreading the token burst across ~28 seconds rather than spiking it all at once.
    🔄
    Retry with exponential backoff — On 429 (rate limit) or 529 (overload), the agent waits 15s → 30s → 60s between up to 3 retry attempts before giving up.
    ✂️
    Context truncation per persona — After relevance filtering, each Claude agent's context is capped at its persona-specific limit (50K–100K chars ≈ 12K–25K tokens), keeping individual prompt sizes within the rate limit window.
    🔀
    Graceful Gemini fallback — If a Claude agent exhausts all retries, it silently switches to Gemini with live Google Search grounding rather than returning an error. The user sees a completed report regardless.

    🎯

    Synthesis — "The Verdict"

    Claude Sonnet 4.6 + Extended Thinking

    After all 18+ agents complete, a sequential synthesis call reads every report concatenated and produces the master action plan. This is the only step that knows about all other agents — it resolves contradictions, identifies consensus (findings validated by 3+ independent agents), fills blind spots, and produces three distinct strategic paths forward: Conservative, Balanced, and Transformative — each with specific tools, timelines, and investment levels tailored to this codebase. The synthesis output then triggers Phase 5 (Living Documentation) and Phase 6 (Dynamic Agent Spawning) automatically.

    Extended Thinking is enabled with an 8,000-token private reasoning budget. Before writing a single word of the final report, Claude reasons through the contradictions in a private scratchpad — e.g., "Performance says cache aggressively; Security flags caching as a risk vector — which wins and why?" The thinking blocks are discarded before storing the output. Only the authoritative final answer is shown. Output budget is 10,000 tokens.

    ⚡ Situational Opus escalation: when the confidence pre-flight (Phase 2.5) detects a difficult run — 4+ agents at low confidence, ≥30% of the fleet at low confidence, or zero high-confidence anchors — synthesis is automatically escalated to Claude Opus 4.6 with a larger 16K thinking budget and 12K output budget. Opus costs ~5× Sonnet but reasons more deeply through ambiguous evidence and resolves harder contradictions. The escalation reason is shown in the synthesis card so you know why we paid more.

    Living Documentation Generation

    Gemini 2.0 Flash (free tier)

    After synthesis completes, the engine automatically generates or incrementally updates six living documents for the project. These aren't static snapshots — each document is aware of its previous version and evolves with every run.

    A single Gemini Flash call receives: the current synthesis, all agent reports, any existing versions of each document, and the project metadata. It produces all 6 documents in one pass — effectively free on the Gemini free tier. Documents are stored as project artefacts in Supabase and are available from the project's Documents tab.

    📊 Run Summary Executive snapshot of this run — key findings, agent consensus, what changed since last time. Your quick-reference card.
    📝 Lessons Learned Cumulative knowledge: what worked, what didn't, recurring patterns, and recommendations that agents keep making across runs.
    📋 Decision Log Architecture decisions, technology choices, and trade-offs identified — timestamped and attributed to the agent that flagged them.
    ⚠️ Risk Register Living risk register: new risks added, existing risks re-scored, mitigated risks closed. Cross-referenced with Security, Compliance, and DevSecOps findings.
    🔧 Tech Debt Tracker Every piece of tech debt flagged by any agent, scored by severity, effort to fix, and business impact. Tracks whether debt is growing or shrinking.
    🤖 Agent Notes Internal agent observations that don't fit other categories — anomalies, gut instincts, "something feels off about this module" notes for future runs.

    Dynamic Agent Spawning

    Gemini Flash + Claude Sonnet

    After synthesis, the engine performs automated gap analysis: did the existing fleet miss something important? If it detects coverage gaps — say, the repo has a complex event-driven architecture but no Event Sourcing specialist, or heavy ML pipelines but no MLOps agent — it proposes up to 3 new specialist agents.

    You see the proposals with their name, focus area, and why the gap was detected. Check the ones you want, dismiss the rest. Approved agents are created via a two-pass persona generation: Gemini Flash drafts the specialist's full persona (identity, mission, investigation checklist, deliverables, research mandate), then Claude Sonnet reviews and refines it for rigour. Cost: ~$0.02-0.05 per agent. Created agents persist per project and are available for all future runs.

    Project-scoped Custom agents belong to the project they were created for. They appear in the fleet automatically on every subsequent run.
    Borrowable Other projects can see and borrow specialists from sibling projects — a "GraphQL Specialist" created for Project A can be pulled into Project B.
    Two-pass quality Flash drafts fast and cheap; Sonnet reviews for depth, accuracy, and proper investigation structure. Best of both models.
    Re-run integration After specialists are created, you're offered a re-run. All agents — original + new specialists — analyse together, benefiting from cross-agent briefing.

    🔭 THE FORWARD-THINKING MANDATE — EVERY AGENT, EVERY REPORT

    Traditional discovery tools only tell you how to fix what you have. This engine asks a harder question: should you build it at all?

    Every one of the 16 parallel agents carries an explicit forward-thinking technology directive embedded in their prompt. They are not allowed to default to traditional approaches without first considering whether an AI tool, low-code platform, or intelligent automation could achieve the same outcome faster and cheaper.

    🤖
    AI Coding Tools

    Every agent considers where GitHub Copilot, Cursor, Codeium, or AI-powered code review would have the highest ROI on this specific codebase — not generically, but tied to what they actually found.

    ⚡
    Low-Code Platforms

    Agents actively assess whether components could be replaced by Retool, Appsmith, Bubble, Webflow, FlutterFlow, or n8n — and are honest about when low-code trades control for speed, and when that trade is worth making.

    🔗
    AI Automation

    Manual workflows, repetitive scripts, and hand-rolled integrations are evaluated against AI automation platforms: n8n, Make, Zapier AI, Activepieces. If a workflow can be automated in days instead of months, the agent says so.

    🏗️
    AI-Native Infrastructure

    Agents consider modern AI-native infrastructure: Modal for serverless GPU, Replicate for model inference, Together AI and Groq for fast LLM hosting, Supabase Edge Functions for zero-config deployment.

    Tiered Recommendations — Every Major Suggestion Comes in Three Flavours
    Traditional

    The proven, well-understood approach. Recommended when the team has deep expertise in it, the risk of adoption is too high, or vendor lock-in is unacceptable.

    →
    AI-Augmented

    The same pattern, supercharged. AI coding assistants, AI-powered testing, AI-driven observability. Same team, same stack — significantly higher output.

    →
    AI-Native

    Redesigned from first principles for an AI-first world. Fewer engineers, faster cycles, lower operational overhead — but higher adoption risk and new skill requirements.

    🎙️ THE UNSCRIPTED DEBATE — AGENTS THAT CHALLENGE EACH OTHER

    The meeting room isn't a polished keynote. It's a boardroom where 19 experts with real findings and genuine disagreements sit around the same table — including the OutSystems specialists who present the platform case and face direct challenges on lock-in, developer experience, and whether the platform fits this codebase's actual complexity.

    What makes it unscripted
    • Grounded in real findings — Every speaker cites a specific finding from their own report: a file path, a CVSS score, a latency measurement, a cost estimate. No generic platitudes allowed.
    • Reactive, not monologues — Each agent must directly respond to what the previous speaker said. The debate flows like a real meeting, not a series of prepared statements.
    • Genuine concessions — Agents are allowed and expected to update their position when the other side has better evidence. "That's a fair point about X, but what you're not accounting for is..."
    • Unresolved tensions stay unresolved — Not everything is neatly settled. The debate surfaces real disagreements for the Synthesis agent to adjudicate — not paper over.
    The four conflict arcs
    1
    Speed vs. Safety — An agent pushing for velocity clashes with one demanding security or quality gates. Both have evidence from the same codebase.
    2
    Build vs. Buy vs. AI vs. Low-Code — The AI Innovation Scout asks "does this need to exist at all?" while the OutSystems specialists ask "should this be on a platform rather than hand-coded?" Both challenge the same traditional build assumptions from different angles. Synthesis resolves the arc — or declares it genuinely undecided.
    3
    Investment appetite — Conservative incremental improvement vs. transformative investment. Different risk tolerances, both defensible with real numbers.
    4
    The unexpected alliance — Two agents who seem to be in opposition find common ground, or an agent concedes a point they initially dismissed. Keeps the debate honest.
    Synthesis closes the debate

    The Synthesis agent speaks last and makes binding decisions on every unresolved conflict. It names the three things that must happen first, is direct about what was not resolved and why, and assigns clear ownership to each outstanding tension.

    🧠 MEMORY & LEARNING — THE ENGINE GETS SMARTER EVERY RUN

    Most analysis tools give you a snapshot. This engine builds a relationship with your codebase — remembering what it found, tracking what changed, and evolving its team of specialists to match your project's actual needs.

    🔄
    Run-Over-Run Learning

    Each analysis run stores per-agent findings, the synthesis verdict, and any user Q&A answers. On the next run, every agent receives a structured summary of what was found before — they stop repeating known issues and focus on what changed. By the 3rd–5th run, reports are dramatically more focused.

    📋
    Living Documents

    Six document types are generated after every run and incrementally updated — not overwritten. The Risk Register adds new risks and re-scores existing ones. The Lessons Learned accumulates institutional wisdom. The Decision Log timestamps every architecture choice. Think of them as a project's living memory.

    🔬
    Self-Evolving Fleet

    After every synthesis, the engine examines what was not covered well enough. Found a complex event-driven architecture with no CQRS specialist? Heavy ML pipelines with no MLOps expert? It proposes specialist agents — created via two-pass AI generation (Flash drafts, Sonnet reviews) and permanently added to your project's fleet.

    💡
    Confidence-Driven Quality

    Before agents spend expensive tokens on full analysis, they do a cheap self-assessment. Agents that are confident share preliminary findings with uncertain ones — like a senior colleague briefing a new hire before a meeting. You can also answer their questions directly, giving them ground truth instead of guesswork.

    How Memory Flows Through the Pipeline
    Run 1 Cold Start

    No prior knowledge. Agents work from codebase + recon only. Full discovery mode. Living documents created for the first time.

    →
    Run 2–3 Building Knowledge

    Agents see prior findings. Specialists may be spawned. Documents update incrementally. Analysis shifts from "what is this?" to "what changed?"

    →
    Run 5+ Deep Intelligence

    Rich episodic memory. Specialist fleet tailored to this project. Living docs are comprehensive. Agents focus on delta analysis and trend detection.

    📚
    Living Documentation

    Six document types — Run Summary, Lessons Learned, Decision Log, Risk Register, Tech Debt Tracker, Agent Notes — are automatically generated after every run and incrementally updated. Browse them in the project's Documents tab.

    🔬
    Dynamic Agent Spawning

    After synthesis, the engine detects coverage gaps and proposes up to 3 specialist agents. Approve with a click — they're created via two-pass generation (Flash drafts, Sonnet reviews) for ~$0.02-0.05 each. They persist per project and join every future run.

    💰
    Budget-Aware Recommendations

    Set a budget range, timeline, and path preference (Conservative/Balanced/Transformative) on your project. Every agent receives this context and tailors their recommendations accordingly — no more "$2M rebuild" proposals when you have a $50K budget.

    ⚡
    Situational Opus Escalation

    When confidence probes flag a difficult run (≥4 agents at low confidence, ≥30% of fleet struggling, or zero high-confidence anchors), synthesis is auto-bumped from Sonnet 4.6 to Opus 4.6 with larger thinking + output budgets. ~5× cost when it fires; the synthesis card shows you why.

    🖼️
    Image Vision

    Uploaded screenshots, wireframes, ER diagrams, and whiteboard photos are run through Gemini-vision and converted to a structured 5-section summary (What this is / Visible content / Inferred purpose / Notable details / Open questions) — agents who can't see images get readable provenance-tagged context.

    🔥
    Cross-Domain Flag Routing

    Agents emit a structured CROSS-DOMAIN FLAGS section listing 0-3 findings that critically affect other domains. Flags are parsed and routed into synthesis as binding items the verdict MUST resolve — turning the parallel fleet into a real conversation, not a one-way report dump.

    ✅
    Closed-Loop Flag Resolutions

    Synthesis writes a dedicated "Cross-Domain Flag Resolutions" section. The frontend parses it back, matches each ruling to the original flag (3-pass fuzzy matching), and decorates the flag panel with green ✓ RESOLVED badges showing ruling + assigned owner.

    📊
    Agent Effectiveness Scoring

    After synthesis, every agent gets a 0-100 score based on weighted citation count across 9 sections (Consensus ×5, Flag Resolutions ×4, Critical Path ×3, Blind Spots ×−1 penalty). Each report card gets a verdict badge (★★★ heavy weight / ○ not cited). Custom specialists get a rolling history so you can decide whether to retire low performers.

    🎙️
    Audio Transcription

    Uploaded audio (mp3/wav/m4a/ogg/flac/webm/aac/aiff) is transcribed and summarised by Gemini's native audio understanding. 6-section structured output: transcript with speaker turns, key topics, action items / commitments, open questions, notable quotes — meetings and voice memos become first-class agent context.

    📡 REAL-TIME STREAMING ARCHITECTURE

    Results stream to your browser the moment each agent completes — you don't wait for all 18 to finish before seeing anything. The SSE stream also carries confidence reports, memory status, and specialist proposals.

    Backend (FastAPI)
    status — Fleet lifecycle (cloning, memory_loaded, launching, complete)
    agent_update — Per-agent status (Analyzing... / Rate limited — waiting 15s...)
    agent_result — Full report {persona, name, emoji, status, content}
    confidence_report — Pre-flight confidence levels for all agents
    awaiting_answers — Fleet paused, waiting for user Q&A answers
    specialist_proposals — Post-synthesis specialist agent proposals
    synthesis_escalated — Synthesis bumped from Sonnet → Opus due to low confidence
    cross_domain_flags — Inter-agent flags raised; synthesis must resolve each one
    flag_resolutions — Synthesis ruling + owner mapped back to each flag
    agent_effectiveness — Per-agent score 0-100 measuring synthesis citation weight
    usage_summary — Token counts, costs, model breakdown
    error — Structured error with message
    →
    Frontend (Vanilla JS)
    Progress bar + fleet status message updated
    Per-agent card spinner and sub-status text updated live
    Report card rendered with persona-specific renderer
    Confidence panel shows agent cards sorted low→high with Q&A form
    Fleet pauses — user answers questions or skips to proceed
    Proposal cards with checkboxes — approve or dismiss
    Cost breakdown card rendered below synthesis
    Error displayed inline, fleet continues for other agents

    The SSE connection uses sse-starlette on the backend and the browser's native EventSource API on the frontend. No WebSocket handshake overhead, no polling. The fleet session mechanism uses asyncio.Event to pause the SSE generator during confidence Q&A — the connection stays alive while waiting for user input, then resumes when answers arrive or the user skips.
